# Felipe Ascari

Backend engineer, mostly Go, 15+ years in.

Started out in logistics and municipal government systems, writing a lot of Java. That context forced me to care early about correctness and data consistency, the type of problems where a bug isn't just a bug. Moved into cloud infrastructure from there, AWS and Azure, then eventually into fintech and crypto. Spent a good portion of that time working on financial systems involving stable cryptocurrencies and cashback flows. That exposure got me curious about blockchain and Web3 from an engineering angle, and Go fits that space well.

Most of what you'll find here started as a question I had while working on something real.

**How does this work? Could this be simpler? What happens when it breaks at scale?**

*Each one is a proof of concept, a way to isolate something I kept running into at work, outside the constraints of a production codebase.*

---

## Areas of interest

- Distributed systems and coordination
- Concurrency and synchronization in Go
- Blockchain, Web3 and crypto infrastructure
- Event-driven architectures and async messaging
- Clean architecture and explicit system boundaries
- Backend APIs and high-throughput services

## Featured projects

Each one is a POC built around something specific I hit in production. The idea is to isolate the problem, test the approach, and document what I found.

| Project | What it explores |
|---|---|
| [cashback-platform](https://github.com/fascari/cashback-platform) | Cashback platform in Go using Web3/crypto concepts, clean architecture and DDD |
| [tcp-message-processor](https://github.com/fascari/tcp-message-processor) | Stateful TCP system with job distribution, cryptographic validation and async stats |
| [exchange-crypto-service-api](https://github.com/fascari/exchange-crypto-service-api) | Service API for crypto exchange operations and integrations |
| [kv-store](https://github.com/fascari/kv-store) | Key-value store focused on correctness and internal behavior |
| [go-runtime-demo](https://github.com/fascari/go-runtime-demo) | Go runtime and blockchain concepts, intentionally kept simple |
