# Welcome

I'm a backend engineer mostly working with Go and distributed systems.

This profile is a collection of projects I use to explore backend engineering topics, mainly around Go and distributed systems.

Most repositories here are focused experiments where I try to understand how systems behave under concurrency, failure, and real-world constraints. I use them to document trade-offs, test architectural ideas, and turn concepts from production work and technical reading into concrete code.

---

## Areas of interest

- Distributed systems and coordination
- Concurrency and synchronization in Go
- Clean architecture and system boundaries
- Backend APIs and services
- Blockchain and Web3 (enthusiast)
